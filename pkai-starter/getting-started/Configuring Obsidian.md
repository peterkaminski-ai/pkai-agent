# Configuring Obsidian

Each vault in Obsidian has its own independent settings, plugins, and appearance. This is a little surprising at first — settings you configure in one vault don't carry over to another — but it means each vault can be customized for its purpose.

Here are the recommended settings for any new vault you create.

## How to Open Settings

Click the **gear icon** in the lower-left corner of the Obsidian window. Make sure you're in the right vault before changing settings — the vault name is shown in the bottom-left corner.

## Recommended Settings

### Editor

- **Show line number** — Turn this **on** (under "Display"). Line numbers make it easier to talk about specific parts of a file with Claude Code or collaborators.

### Files and Links

- **Detect all file extensions** — Turn this **on**. This lets you see all file types in the file browser, not just Markdown files. (Obsidian calls this "Show all file types" in some versions.)
- **Default location for new notes** — Set to **"Same folder as current file."** This keeps new notes near related content instead of dumping everything in the vault root.

### Appearance

- **Inline title** — Turn this **off** (under "Interface"). This prevents Obsidian from displaying a large title at the top of every note, which can be redundant when you already have the file name in the tab.

## A Note on Per-Vault Settings

If you later create additional vaults, you'll need to configure these settings again for each one. It feels tedious, but it only takes a minute, and it means your vaults stay independent.
